# Protocol Route File: Exploratory

## What This Stage Protects
- The search-to-claim boundary.
- Hypothesis-generating status for surfaced patterns, structures, and anomalies.
- Reproducible search posture without turning exploration into fake confirmation.

## What This Stage Prohibits
- Letting open search results masquerade as confirmed evidence.
- Creating partitions whose real function is to launder searched patterns into validated findings.
- Letting later stages convert exploratory outputs into inferential, predictive, or causal claims without reopening upstream.

## What This Stage Defers
- Exact search procedures, algorithm families, parameter grids, subgroup definitions, anomaly thresholds, embedding choices, and the concrete analysis specification.
- Any confirmatory or predictive validation program.

## What Triggers Backtracking In This Stage
- A candidate pattern depends on one arbitrary encoding, one subgroup definition, one algorithm family, or one narrow parameter range.
- The work starts needing unseen-data performance claims, population generalization, or causal interpretation.
- Grouping, temporal dependence, or interference matters more than protocol allowed.
- Candidate patterns cannot be specified reproducibly enough for later analysis.
